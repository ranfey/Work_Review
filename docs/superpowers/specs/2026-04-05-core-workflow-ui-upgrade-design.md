# 核心工作流 A 包 UI 升级设计

## 背景

时间线与时段摘要页已经升级为 `A1 纸感编辑部` 风格，但主工作流中的其它关键页面仍然保持原有的通用后台式视觉语言。当前产品在页面间跳转时，存在明显的风格断层：侧边栏是导航底座，概览是数据首页，日报是阅读页，助手是分析页，但四者还没有形成统一的界面系统。

用户已确认本轮先做 `A 包`：

1. `Sidebar`
2. `Overview`
3. `Report`
4. `Ask`
5. 对 `Settings / About` 做轻量统一

## 目标

1. 让核心工作流四页共享统一的纸感编辑部视觉语言。
2. 保持每个页面的职责差异，而不是复制相同布局。
3. 提升产品整体完成度，让页面切换不再像进入不同应用。
4. 不改动后端接口与数据语义，控制风险在展示层。

## 非目标

1. 不在本轮重构 `Settings` 与 `About` 的核心信息结构。
2. 不重构概览、日报、助手的后端生成逻辑。
3. 不重做路由系统或应用壳层。
4. 不把四个页面强行统一成同一种版式。

## 统一视觉方向

整体继续沿用 `A1 纸感编辑部`：

1. 纸感暖白背景与深墨色正文。
2. 琥珀 / 靛蓝作为受控强调色。
3. 强调整体秩序、呼吸感、章节感。
4. 避免通用 dashboard 风格和社交气泡风格。

## 页面策略

### Sidebar

侧边栏是导航基座，只做轻改，但必须改精。

目标：

1. 让导航更像目录栏，而不是普通列表。
2. 让激活态更有秩序和焦点感。
3. 让品牌区、录制状态区、底部工具区形成统一层次。
4. 品牌副标题不再依赖换行文本，而是改为更稳定的标签式信息。
5. 导航项不保留装饰性编号，避免窄侧边栏中出现额外噪音。

原则：

1. 侧边栏不抢主内容风头。
2. 优先提升精度、留白和秩序感。
3. 不引入复杂动画或额外控件。

### Overview

概览页改成“今日工作总编台”。

目标：

1. 顶部先呈现日期、状态和当天工作概览。
2. 统计卡片形成主次层级，不再所有模块同权。
3. 网站访问、应用使用、小时分布形成章节感更强的内容分区。

原则：

1. 保持高信息密度和高效率。
2. 通过章节与节奏取代“卡片平铺”。
3. 不做大屏炫技展示。

### Report

日报页改成“纸感成稿页”。

目标：

1. 让日报更像正式成稿，而不是 Markdown 容器。
2. 强化标题、日期、生成状态与内容章节的阅读层级。
3. 保留导出和生成操作，但将操作退到内容之外。

原则：

1. 页面主角是正文。
2. 控件应该是页边工具，不应干扰阅读。
3. Markdown 层级需要更强的章节感和纸面感。

### Ask

助手页改成“工作研究台”。

目标：

1. 让用户明确知道这里是基于工作记录的问答界面。
2. 将对话、证据、输入区组织成更像研究台的结构。
3. 降低“通用聊天页”既视感。

原则：

1. 保持对话流畅，不做过重的双栏工作站。
2. 引用卡片要更像证据块。
3. 首屏空状态要更明确地告诉用户可提问范围与数据依据。

### Settings / About 轻量统一

这两页不做重结构改造，但要补进统一系统：

1. `Settings` 强化页面壳层与保存操作区，保持清晰优先。
2. `About` 强化封面卡和信息分组，让它不再跳出同一套视觉语言。
3. 设计强度低于 `Overview / Report / Ask`，避免次要页面抢戏。

## 实现边界

### 不改动的数据链路

继续复用：

1. 概览统计与网站访问数据
2. 日报读取、生成、导出
3. 助手消息、引用与模型选择
4. 侧边栏语言切换、主题切换和录制状态

### 允许新增的前端结构

1. 页面级语义容器与章节区块
2. 更细分的局部样式类
3. 少量新的展示文案或辅助标签
4. 针对布局结构的源码测试

## 文件范围

预计改动文件：

1. `src/lib/components/Sidebar.svelte`
2. `src/routes/Overview.svelte`
3. `src/routes/report/Report.svelte`
4. `src/routes/ask/Ask.svelte`
5. `src/routes/settings/Settings.svelte`
6. `src/routes/about/About.svelte`
7. `src/app.css`
8. 新增对应布局测试文件

## 测试策略

至少覆盖以下验证：

1. 侧边栏存在新的编辑部导航结构。
2. 概览页存在新的总编台式分区结构。
3. 日报页存在新的成稿容器和阅读布局结构。
4. 助手页存在新的研究台结构和证据卡布局。
5. 设置页与关于页接入统一壳层且不破坏现有测试。
6. 现有 locale、路由、概览、日报测试继续成立。
7. 前端构建通过。

## 验收标准

满足以下条件即可认为 A 包达标：

1. 四个页面看起来属于同一产品系统。
2. 页面间职责区别更清楚，不再只是样式不同。
3. 概览更像总控台，日报更像成稿，助手更像研究台。
4. 侧边栏更精致，品牌区更稳，导航不再出现多余编号。
5. 设置 / 关于 进入同一视觉系统，但不抢主工作流页面风头。
6. 现有功能与验证保持可用。
