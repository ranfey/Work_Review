# 时间线编辑部轨道改版设计

## 背景

当前时间线页面采用“普通日志列表”形态：时间、应用图标、标题、时长按一行展开。时段摘要页则是“按小时卡片 + bullet 摘要”的旧结构。信息足够完整，但视觉节奏单一，难以体现“回看一天工作轨迹”的产品价值。用户希望时间线更有辨识度，同时保持描述清晰，不要堆砌过多说明文案，并要求时段摘要页同步升级。

## 目标

1. 将时间线从普通列表升级为可回看的叙事型轨迹。
2. 保持信息读取效率，避免为了视觉效果牺牲清晰度。
3. 用少量重点截图卡片强化记忆点，而不是让所有记录都露图。
4. 让时段摘要页和主时间线共享同一套纸感编辑部视觉语言，但保持不同的信息密度和页面职责。
5. 不改动后端接口和数据语义，尽量将风险控制在前端展示层。

## 非目标

1. 不重构 `get_timeline`、`get_activity`、`get_hourly_summaries` 的接口。
2. 不改动记录采集、聚合、分类回填、截图存储逻辑。
3. 不顺带重做除 `Summary.svelte` 之外的其它页面。
4. 不引入复杂的“智能重点识别”模型，仅采用轻量可解释规则。

## 设计方向

### 整体气质

采用 `A1 纸感编辑部` 方向：

1. 视觉基调为米白纸感背景、墨黑正文、琥珀强调色。
2. 页面像“工作轨迹档案”，不是传统 dashboard，也不是炫技式 scrollytelling。
3. 动效克制，只承担引导理解的作用。

### 页面骨架

时间线列表改为双区结构：

1. 左侧为强时间主轴。
   - 每条记录都与时间点绑定。
   - 时间点和轨道线成为阅读锚点。
2. 右侧为轨迹卡片区。
   - 普通记录为紧凑卡片。
   - 重点记录为放大卡片。
3. 窄窗口下仍保持纵向单列，只缩窄时间区，不取消时间主轴。

### 时段摘要页骨架

`Summary.svelte` 改为“阶段带状版”：

1. 页面头部与主时间线统一为纸感编辑部风格。
2. 主体使用按小时排列的带状区块，而不是普通卡片堆叠。
3. 每个区块左侧显示小时锚点与该小时累计时长。
4. 右侧显示一句主摘要和 2-4 个应用标签。
5. 少数高活跃时段允许有轻量强调，但不和主时间线争夺主视觉。

### 卡片层级

#### 重点卡片

重点卡片只占少数，用于承载“这一段你在做什么”的记忆锚点。

结构：

1. 展示截图缩略图。
2. 展示应用图标、应用名、分类标签、时长。
3. 展示精简后的窗口标题，最多两行。
4. 可选展示 URL 的简短辅助信息，但不把 URL 变成长段主文案。

#### 普通卡片

普通卡片保持高密度和高可扫读性。

结构：

1. 应用图标 + 应用名
2. 单行标题
3. 时长
4. 必要时的分类标签

普通卡片默认不展示大截图，避免页面噪音过高。

### 时段摘要信息层级

时段摘要页与主时间线分工不同：

1. 主时间线负责逐条回放。
2. 时段摘要负责按小时快速总览。
3. 每个小时只保留一句主摘要，不再渲染 bullet 列表。
4. 应用标签仅保留 2-4 个主要项，避免“信息摘要页”重新变成碎片列表。

### 重点卡片选择规则

重点卡片采用轻规则，要求易懂、可维护、无额外后端依赖：

1. 优先要求存在截图。
2. 时长较长的记录优先。
3. 与前一条相比出现明显阶段切换时可提升优先级，例如应用或分类变化。
4. 需要限制数量和密度，避免连续出现多张重点卡片。

实现层可以用“评分 + 稀疏筛选”的方式，但最终结果必须稳定、可解释。

## 交互设计

1. 列表默认即清晰可读，不依赖 hover 才能理解内容。
2. hover 只做轻微浮起、边框加深、时间点高亮。
3. 点击卡片继续使用现有详情弹层。
4. 详情弹层头部视觉与列表卡片统一，像从主轨道里抽出一张档案卡。
5. 加载更多仍保留，但视觉上和主轨道一致，像轨迹的延续，而不是单独按钮块。
6. 加载、空状态、报错状态统一为同一套纸感视觉语言，文案保持简短。
7. 时段摘要页不做展开折叠，避免把总览页再次做成复杂交互页。

## 暗色模式

暗色模式不做简单反色，而是改为“深纸感”：

1. 背景维持深色，但保留柔和暖调层次。
2. 强调色仍使用偏琥珀或柔和暖色，避免变成通用蓝紫科技面板。
3. 保证时间轴、标题、截图边界在暗色下仍清晰。

## 实现边界

### 数据与逻辑

保留现有能力：

1. 时间线加载、分页、详情获取
2. 缩略图 / 高清图缓存
3. 应用分类修改与历史回填
4. 页面可见性时钟刷新策略

新增前端展示辅助逻辑：

1. 重点卡片判定函数
2. 标题 / 副信息的精简展示函数
3. 新布局所需的局部样式与结构

### 文件范围

预计改动文件：

1. `src/routes/timeline/Timeline.svelte`
2. `src/routes/timeline/Summary.svelte`
3. `src/routes/timeline/TimelineLayout.test.js`
4. `src/routes/timeline/SummaryLayout.test.js`
5. `src/lib/i18n/index.js`

如样式组织需要复用，再考虑是否补充 `src/app.css`，但优先将变更收敛在时间线子页面本身。

## 测试策略

本次改版至少覆盖以下验证：

1. 页面源码中存在新的编辑部轨道结构和重点卡片逻辑。
2. 重点卡片与普通卡片均可点击查看详情。
3. 无截图记录仍会落入清晰的无图展示，而不是错误态。
4. 分类修改入口与相关文案不受影响。
5. 时段摘要页存在新的阶段带状结构、一句主摘要模式和应用标签结构。
6. 现有时间线截图占位、可见性刷新、locale 相关测试继续成立。

## 验收标准

满足以下条件即可认为改版达标：

1. 页面第一眼不再像普通日志列表，而像工作轨迹页面。
2. 用户能快速找到重点记录，同时顺着时间主轴继续扫读。
3. 页面没有变得啰嗦，标题和副信息仍然克制。
4. 时段摘要页和主时间线风格统一，但职责区分清楚。
5. 详情弹层、分类修改、分页加载均保持可用。
6. 相关测试与构建验证通过。
