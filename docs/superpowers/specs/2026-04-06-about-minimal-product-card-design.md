# About 页极简产品名片重构设计

## 背景

当前 About 页已经完成第一轮与第二轮统一，但页面仍处于“既不够简洁，也不够完整”的中间态。它同时承载了品牌介绍、主要动作、数据目录、支持文案与平台信息，导致视觉中心不稳，整体观感像信息拼盘，而不是产品名片页。

用户明确要求：参考当下更流行的产品介绍页趋势，对 About 页进行重构，同时保持与整套应用一致的视觉架构。

## 目标

1. 将 About 页重构为极简产品名片页。
2. 保留必要动作，但明确主次，不再让所有按钮同权。
3. 保留本地数据说明与平台兼容信息，但降为次级内容。
4. 保持与现有 `A1 纸感编辑部` 视觉系统一致。

## 非目标

1. 不把 About 页做成完整官网式长页。
2. 不改动赞助弹层的核心功能与二维码内容。
3. 不重构更新逻辑、打开数据目录逻辑、Linux 会话信息逻辑。

## 设计方向

About 页采用“极简产品名片”结构：

1. **品牌主卡**
   - Logo
   - Work Review
   - 版本号
   - 一句产品说明

2. **主操作条**
   - `检查更新` 作为主按钮
   - `GitHub` / `打开数据目录` 作为次按钮
   - `赞助支持` 作为柔和入口

3. **可信说明卡**
   - 本地优先
   - 时间线回看
   - 日报生成
   每块只保留短标题与一句说明。

4. **系统说明卡**
   - 本地数据目录
   - 运行环境或 Linux 会话兼容信息（仅在需要时）

## 版式策略

1. 主视觉必须聚焦在品牌主卡。
2. 页面不再使用平均四宫格结构。
3. 动作条与说明卡分离，不混成单一拼盘。
4. About 页的留白比设置页更大，阅读密度更低。

## 风格约束

1. 继续使用纸感暖白背景与深墨色正文。
2. 保留柔和高光与边框，但降低花哨感。
3. 技术栈 chip 不再占据主视觉，可弱化为附加信息。

## 文件范围

预计改动文件：

1. `src/routes/about/About.svelte`
2. `src/app.css`
3. `src/routes/about/AboutEditorial.test.js`

## 测试策略

1. 新结构存在：品牌主卡、主操作条、可信说明卡、系统说明卡。
2. 旧结构中的多宫格式内容分组被替换。
3. 现有 About 页功能测试继续通过。
4. 构建通过。
