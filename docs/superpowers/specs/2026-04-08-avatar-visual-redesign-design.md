# 桌宠视觉升级设计

## 目标

将当前桌宠从“单层线稿猫 + 少量表情切换”升级为“分层矢量职场搭子猫”，让它在保持轻量实现的前提下，具备更强的角色识别度、状态差异感和桌面陪伴感。

## 当前问题

1. 角色轮廓单薄，头身比例和四肢语言不够明确，缩小时辨识度不足。
2. 视觉变化主要集中在眼睛和嘴巴，状态切换更多像图标变化，不像活体角色。
3. 主体只有单层填充和描边，缺少脸部、肚皮、尾巴尾尖、阴影等层次。
4. 气泡已经具备较成熟的质感，但桌宠本体仍偏“早期草图”。
5. 当前动作更多是整体浮动，缺少耳朵、尾巴、头部、视线的细粒度节奏。

## 设计方向

采用“轻治愈办公猫”方向，保留现有白猫认知，但升级为更现代的 Q 版职场搭子形象。

### 视觉关键词

- 大头小身
- 柔和深描边
- 低饱和暖白毛色
- 脸部和肚皮分层
- 尾巴有情绪
- 状态配件轻量点缀
- 动作幅度小但持续有生命感

### 角色设定

- 物种：白色短毛办公猫
- 性格：安静、机灵、可靠，不过度打扰
- 定位：工作状态提示器 + 陪伴式反馈角色
- 情绪表达：通过耳朵、尾巴、眼型、腮红和小配件表达，而不是只靠文案

## 架构约束

1. 保留当前 `Svelte + SVG + CSS 动画` 技术路线，不引入 Live2D、Canvas 绘制或外部运行时。
2. 保留现有状态驱动模型，由 `mode + contextLabel + motionBeat` 决定视觉表现。
3. 不改动桌宠窗口事件链路、Tauri 拖拽机制和状态事件订阅逻辑。
4. 新增视觉层时优先复用现有组件边界，避免引入新的渲染组件树。

## 形象拆解

新的桌宠主体由以下分层构成：

1. 身体底层：主体毛色、肚皮、尾巴、爪垫投影。
2. 头部层：头型、耳朵、脸颊、鼻口区。
3. 眼部层：眼白、瞳孔、高光、眼睑。
4. 情绪层：腮红、眉线、半闭眼、惊讶眼、专注眼。
5. 配件层：耳机、眼镜、围巾、光环、工作徽标等轻量装饰。
6. 环境层：脚下阴影、状态微光。

## 状态策略

### 核心状态

- `idle`：自然呼吸，眼神巡视，尾巴轻摆。
- `working`：专注眼，更紧的耳位，动作节拍略快。
- `reading`：眼睑微收，头部下探，尾巴节奏更平缓。
- `meeting`：姿态更端正，耳机或提示点点缀。
- `music`：头部轻点拍，尾巴更有律动。
- `video`：视线更集中，惊讶幅度降低。
- `generating`：高光和轮廓微发亮，带思考/扫描感。
- `slacking`：更松弛的耳位与嘴型，身体略下沉。

### 上下文变体

在大状态基础上保留 `contextLabel` 变体机制，但从“嘴巴/尾巴少量替换”升级为：

- 表情参数
- 配件显隐
- 视线方向
- 尾巴情绪等级
- 动作节拍模板

例如：

- `编码中`：眼神更聚焦，左右爪节奏交替更紧，尾巴控制更稳。
- `调研中`：视线轻偏移，尾巴更活跃，耳朵外扩。
- `演示中`：嘴型更开，头部抬起，提示点更明显。
- `生成中`：眼部高光增强，状态微光出现。

## 视觉规范

### 轮廓

- 头部占比明显增加，整体接近 60% 视觉重心。
- 身体更短、更圆，减少“站立小人感”。
- 耳朵更大、更尖，成为远距离识别点。
- 尾巴更长并保留明显尾尖变化，承担情绪表达。

### 色彩

- 主体采用暖白色，不使用纯白描边对比。
- 描边从当前高对比深灰改为更柔和的蓝灰墨线。
- 每个状态保留一组点缀色，但只作用于耳内、腮红、配件、微光。
- 背景保持透明，不引入大面积底板。

### 动效

- 保留整体浮动，但减弱机械感。
- 增加随机眨眼、耳朵轻抖、尾巴回弹、头部短停顿。
- 状态切换前后增加轻微蓄力与回落，而不是立即换脸。

## 交互策略

1. 单击拖拽逻辑保持不变。
2. 双击打开主界面逻辑保持不变。
3. 气泡锚点可根据新头部比例微调，但维持现有紧凑悬浮样式。
4. 暂不引入复杂桌面物理行为，优先提升视觉可信度。

## 实施边界

本轮只做“角色重塑 + 状态表现增强 + 少量配件层”，不做以下内容：

- Live2D / Spine / Rive 集成
- 资源下载或工坊系统
- 桌面碰撞、吸附、重力模拟
- 多角色切换
- 新设置项扩展

## 测试要求

1. 轮廓测试需验证新分层路径存在，且旧的单层假设已被替换。
2. 状态元信息测试需覆盖新 appearance token、配件开关和状态差异。
3. 组件源码测试需验证新增结构类名和视觉层已渲染。
4. 最终至少执行桌宠相关测试集合与一次构建验证。
